What is Kali Linux?

  • A Debian-based Linux distribution designed for digital forensics and penetration testing
  • Maintained and funded by Offensive Security
  • Comes pre-installed with over 600 penetration-testing programs
  • Widely used by security professionals and ethical hackers

Key Features of Kali Linux

  • Pre-installed security and penetration testing tools
  • Supports wireless device injection
  • Custom kernel patched for injection
  • Developed in a secure environment
  • GPG signed packages and repositories

Common Kali Linux Tools

  • Nmap - Network mapping and port scanning
  • Wireshark - Network protocol analyzer
  • Metasploit - Penetration testing framework
  • John the Ripper - Password cracking
  • Burp Suite - Web application security testing
  • Aircrack-ng - Wireless network security

Kali Linux Command Line Basics

Essential commands for security tasks:

# Update package lists and upgrade system
sudo apt update && sudo apt full-upgrade -y

# Search for security tools
apt search kali-tools-

# Install a specific tool (e.g., nmap)
sudo apt install -y nmap

# Run a tool with root privileges
sudo nmap -sV [target]

# Start the Metasploit Framework
msfconsole

Introduction to Shell Scripting

  • Automate repetitive tasks in Kali Linux
  • Combine multiple commands into scripts
  • Create custom tools and utilities
  • First line typically: #!/bin/bash
  • Make scripts executable with: chmod +x script.sh

Shell Scripting Basics

#!/bin/bash
# This is a comment
VARIABLE="Hello, World!"
echo $VARIABLE

# If statement
if [ -f "file.txt" ]; then
    echo "File exists"
fi

# For loop
for i in {1..5}; do
    echo "Number: $i"
done

Advanced Shell Scripting

Functions and Parameters:

#!/bin/bash
# Function definition
greet_user() {
    echo "Hello, $1!"
}

# Function call with parameter
greet_user "Kali Linux User"

# Using command line arguments
echo "First argument: $1"
echo "Total arguments: $#"
echo "All arguments: $@"

# Exit codes and logical operators
command1 && echo "Command1 succeeded" || echo "Command1 failed"
command2 || { echo "Command2 failed"; exit 1; }

Text Processing with AWK and SED

AWK examples:

# Print first column of a file
awk '{print $1}' file.txt

# Print lines where the 3rd field is greater than 100
awk '$3 > 100 {print $0}' data.txt

# Using field separators
awk -F':' '{print $1, $6}' /etc/passwd

SED examples:

# Replace text in a file
sed 's/old/new/g' file.txt > newfile.txt

# Delete lines containing 'pattern'
sed '/pattern/d' file.txt

# In-place editing
sed -i 's/foo/bar/g' file.txt

Practical Scripting Projects

Log Analyzer:

#!/bin/bash
# Analyze web server logs
echo "Top 10 IP addresses:"
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -nr | head -10

echo -e "\nRequests by status code:"
awk '{print $9}' /var/log/nginx/access.log | sort | uniq -c

File Organizer:

#!/bin/bash
# Organize files by extension
for file in *; do
    if [ -f "$file" ]; then
        ext="${file##*.}"
        mkdir -p "$ext"
        mv "$file" "$ext/"
    fi
done

Shell Scripting Best Practices

  • Always start with a shebang (#!/bin/bash)
  • Use meaningful variable names and add comments
  • Enable error checking with set -e and set -u
  • Quote all variables to prevent word splitting
  • Use functions to organize code into logical blocks
  • Validate input and handle errors gracefully
  • Use [[ ]] for tests instead of [ ]
  • Follow the ShellCheck guidelines for clean code

Using Gemini CLI for Scripting

Basic usage examples:

# Generate a script to find and archive old log files
gemini generate "Write a bash script that finds log files older than 30 days in /var/log, 
compresses them into a tar.gz archive with timestamp, and deletes the original files"

# Get help with a specific command
gemini ask "How can I recursively find and replace text in multiple files?"

# Explain what a complex command does
gemini explain "find . -type f -name '*.txt' -exec grep -l 'error' {} \; | xargs rm -f"

# Convert commands into a script
gemini scriptify "Find all .jpg files larger than 1MB and copy them to a backup directory"

Best Practices with AI-Generated Scripts

  • Always review generated scripts before execution
  • Test in a safe environment first
  • Be specific in your prompts for better results
  • Use version control for AI-generated scripts
  • Document the purpose and functionality

Debugging and Optimization

  • Use set -x for debugging
  • Check exit codes with $?
  • Time commands with time command
  • Profile scripts with bash -x script.sh
  • Optimize loops - avoid unnecessary commands in them
  • Use built-in string operations instead of external commands

Forensic Scripting Best Practices

  • Always work on copies of evidence, never originals
  • Maintain chain of custody in script outputs
  • Include comprehensive logging in all scripts
  • Validate all tool outputs and hashes
  • Document all assumptions and limitations
  • Test scripts thoroughly in a controlled environment
